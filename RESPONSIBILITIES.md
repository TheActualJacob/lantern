Person A — Depth + Recon (host-side, Python)
Day 1
Hr 1: Clone DA-V2 Small, patch interpolate_pos_encoding to fixed-shape traceable variant, load weights, verify inference on a still image. Use a surrogate RGBD dataset (ScanNet / 7-Scenes) while waiting for C's recordings. Deliverable: depth_model.py — takes (H,W,3) image, returns (H,W) relative disparity map.
Hr 2–3: TSDF integration — wire Open3D VoxelBlockGrid (prototype) or VDBFusion. Test it with ground-truth depth first (not DA-V2 output), so you know the TSDF layer itself is correct before adding the scale solver on top. Deliverable: tsdf_fuse.py — takes (metric_depth, intrinsics, pose) sequence, returns .glb mesh.
Hr 4–5 — the crux: Affine scale/shift solver. Fit metric_disparity ≈ s·pred_disparity + t (both s AND t — the prior draft only solved s, which bends flat surfaces). Do this in disparity/inverse-depth space. Use Huber loss or RANSAC to reject ARCore edge-bleed outliers. Solve one global s across the sequence; allow a small per-frame t, temporally smoothed. Deliverable: scale_solver.py — solve_affine(pred_disp, sparse_metric, conf_map) → (s, t). Include a unit test on synthetic data where the correct [s,t] is known.
Hr 6: Feed scaled depth into TSDF. Validate on 3 real objects from C's recordings. If surfaces look thick or drift between views, escalate: treat ARCore's confident sparse points as the consistent skeleton and use DA-V2 only as a dense guide (guided/joint-bilateral upsampling = depth completion). Deliverable: end-to-end pipeline_float.py — images in, .glb out, working on 3 objects.
Hr 7: Hand .glb outputs to D for ground truth comparison. Document the affine solver's residuals so B has calibration quality context.
What A needs from C by Hr 2: recorded ARCore session files (RGB frames + poses + intrinsics + 16-bit raw depth + confidence). If C isn't ready, use ScanNet as a surrogate but flag it — the calibration domain won't match.

Person B — Edge/QNN (host-side, Python)
Day 1
Hr 1: Install ExecuTorch + QNN SDK. Confirm QcomChipset.SM8750 — the S25 is SM8750, NOT SM8850 (that's the S26). This is the first thing you check. If the enum is missing, mirror the nearest entry and patch locally before doing anything else. Deliverable: env_check.py — imports QnnPartitioner, prints QcomChipset.SM8750 value without error.
Hr 2–3: torch.export DA-V2 Small → to_edge_transform_and_lower with QnnPartitioner(SM8750). Read the fallback report, but the gate is not "100% op coverage" — it's "few contiguous partitions." Count the number of distinct QNN-delegated subgraphs. Many small partitions = per-frame CPU↔NPU boundary thrash = dead latency even at 95% coverage. DA-V2's interpolate/normalize ops habitually land on CPU and fragment the graph. Fix by: patching ops to traceable equivalents, using skip_node_op_set to deliberately leave cheap ops on the portable backend, or restructuring the export. Iterate until you have ≤3 contiguous QNN partitions. Deliverable: lower_depth.py + partition_report.txt — lists every partition, whether QNN or CPU, and the partition count.
Hr 4–6: PT2E quantization. QnnQuantizer() → prepare_pt2e → calibrate on A's frames (or surrogate dataset) → convert_pt2e. Produce both depth_8a8w.pte and depth_16a8w.pte. For calibration, use frames from C's actual capture domain — not random ImageNet-style images. Deliverable: both .pte files committed.
Hr 7: Run each .pte against float on the QNN x86 reference path. Compute the TSDF mesh diff (not just depth MAE) by feeding each variant's output into tsdf_fuse.py and comparing meshes. Deliverable: numerics_report.txt — L1 relative depth error for 8a8w and 16a8w + side-by-side mesh screenshot. Pick the shipping variant.
What C needs from B by end of Day 1: depth_8a8w.pte (or depth_16a8w.pte if 8a8w fails) + numerics_report.txt.

Person C — Android/ARCore (phone + Android Studio)
Day 1
Hr 1–2 — first priority: Build the capture harness. This is not the live inference loop — it's a recorder. Per keyframe, save to disk: RGB frame, Frame.getCameraPose() (ARCore world meters, 6-DoF), CameraIntrinsics, Frame.acquireRawDepthImage16Bits() (16-bit buffer: 13-bit depth + 3-bit confidence). Include a sync validator: confirm timestamps align across streams, interpolate pose to each RGB timestamp (slerp rotation, lerp translation), and log trajectory as a sanity check. Send session files to A and B by end of Hr 2 — this is the most important handoff of Day 1. Deliverable: Android app that records and saves synced ARCore sessions to disk.
Hr 3–4: Integrate ExecuTorch Android runtime with a stub model (same tensor shape as DA-V2, random output). Confirm .pte load → single-frame inference → tensor output works in the app without crashing. Also wire the keyframe-gating logic here as an architectural decision, not an afterthought: only run depth inference when inter-frame translation exceeds a threshold (pure rotation gives zero parallax baseline → scale solve degenerates). Deliverable: app runs stub inference on a live frame and logs output shape; keyframe gate toggleable.
Hr 5–7: Swap stub for real depth_8a8w.pte from B. Single frame on-device → diff output against B's numerics_report.txt. If they match, confirm the runtime is working. Deliverable: on-device inference confirmed; depth output matches host within tolerance.
